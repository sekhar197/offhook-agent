# Core benchmarks

Generated by `npm run bench` — deterministic synthetic workload (seeded),
no network, no API keys. Numbers below are from the machine that last ran
the suite; run it yourself to get yours.

- **Machine:** Apple M5 (10 cores), darwin arm64, Node v22.18.0
- **Date:** 2026-06-12

## Hot-path latency

These run on EVERY caller turn, so they are budgeted in microseconds-to-
single-digit-milliseconds. The voice pipeline's perceived latency is
dominated by STT/LLM/TTS — the core must stay invisible next to them.

| Operation | p50 | p95 | p99 | max |
|---|---|---|---|---|
| search (100 entries, 250 queries) | 219µs | 516µs | 1.12ms | 10.00ms |
| search (1,000 entries, 250 queries) | 1.62ms | 2.06ms | 2.30ms | 2.67ms |
| search (10,000 entries, 250 queries) | 16.77ms | 22.93ms | 24.99ms | 26.85ms |
| resolver (100 entries, 300 lookups) | 8µs | 87µs | 278µs | 472µs |
| resolver (1,000 entries, 300 lookups) | 22µs | 300µs | 721µs | 877µs |
| resolver (10,000 entries, 300 lookups) | 130µs | 644µs | 1.04ms | 1.49ms |
| ASR correction (1,000-entry KB, 1,000 transcripts) | 27µs | 323µs | 717µs | 1.13ms |
| prompt build (80-entry KB, 2,000 builds) | 44µs | 67µs | 147µs | 3.85ms |

Resolver index build (startup / knowledge reload, not per-turn):
- 100 entries: 1.7ms
- 1,000 entries: 5.6ms
- 10,000 entries: 54.6ms

## Retrieval accuracy (golden set, keyword path only)

| Case | hit@1 | hit@3 |
|---|---|---|
| retrieval exact (n=40) | 100.0% | 100.0% |
| retrieval asr-mangled (n=40) | 47.5% | 85.0% |

92.5% hit@3 overall (BM25+rerank only — embeddings add semantic rescue on top).
The tool layer returns max 3 entries to the LLM, so **hit@3 is the number
that decides whether a caller hears the right answer.**

## Stress (10,000-entry knowledge base)

- 2,000 searches over a 10,000-entry KB, sequential: **40.62s** total (20.31ms/query, 49 qps)
- Same load in concurrent bursts of 100: **42.56s** total (47 qps)
- 2,000 resolver lookups on the same KB: **420ms** total (210µs/lookup)
- Heap after stress run: **45 MB**

## What this does NOT cover

End-to-end voice latency (TTFT: caller stops speaking → first audio back)
and conversation quality are properties of the full pipeline and the
deployment's models. Those are measured by the live harness and the
LLM-judged simulated-call evals — see [evals.md](evals.md).
